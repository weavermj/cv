Read Me

Details of how to install, configure and invoke the BBC Pagination Device jQuery plugin are listed here.

Dependencies

  • jQuery - this plugin was written and tested against jQuery version 1.9.1

Installing the plugin

All files relating to the plugin are contained in the BBCPaginate folder. jQuery 1.9.1 is also bundled with the installation files.

The folder structure is as follows:

  • css - contains css specific to the navigational component of the plugin
  • img - contains the previous and next navigation GEL icons which are referenced from the CSS
  • js - contains the plugin file

Download the BBCPaginate.zip file and extract the contents to a suitable location. In the example below we have extracted the contents to a folder 'js' where all other JavaScript libraries are contained.

In your HTML template, you will need to use 3 includes:

<script src="js/lib/jquery-min.js"></script> - jQuery 1.9.1 library, minimised

<link href="js/BBCPaginate/css/pagination.css" rel="stylesheet" media="screen"> - associated CSS for the plugin

<script src="js/BBCPaginate/js/BBCPaginate.jQuery.js"></script> - the BBCPaginate plugin

It is recommended to place the script include for the BBCPaginate.jQuery.js file in the body of your HTML document near to the div elements that you wish to paginate - this will prevent the loading of the script from delaying the rendering of other elements as the HTML page loads. The jQuery and CSS includes can be placed in the HTML <head> section (and you can omit the jQuery include if you already have jQuery configured for your website).

Invoking the plugin

To add the plugin to your HTML page, you will need to add a target div in your HTML source, for example:

<div id="feed1"></div>

To add the pagination device to the <div>, target it with jQuery from a suitable script. I used the following script:

<script src="js/jquery_initiator.js" ></script>

which contains

$("div#feed1").BBCPaginate();

This will render the pagination component in the <div> element that has an id of feed1 with the default options.

Options

  • feed_url - a relative or absolute URL to the file which contains the JSON news items that you wish to paginate

default option: "http://www.bbc.co.uk/news/10284448/ticker.sjson"

To pass in multiple URLs, separate each one with a pipe (|) character, for example:

feed_url: "http://www.bbc.co.uk/news/10284448/ticker.sjson|http://polling.bbc.co.uk/sport/0/16189337/ticker.sjson"

  • feed_type - specifies which format the input feed is in, so that different data formats can be supported

valid options: "static", "bbc_ticker"

default option: "bbc_ticker"

When the feed_type is set to "bbc_ticker", title is mapped to the "prompt" attribute and description is mapped to the "headline" attribute in the JSON data

  • news_type - specifies the type of articles that you wish to filter from the feed(s)

valid options: "all", "breaking"

default option: "all"

  • show_title - control whether or not the title attribute of news items is displayed

valid options: true, false

default option: true

  • update_interval - how quickly, in milliseconds, the JSON news items are refreshed from the feed_url

default option: 60000 (60 seconds)

Example usage

Within the HTML page where paginated news items are required:

<div id="feed_bbc" class="feed"></div>

From a separate JavaScript file: 

    $("div#feed_bbc").BBCPaginate( {
        feed_url: "http://polling.bbc.co.uk/sport/0/16189337/ticker.sjson|http://polling.bbc.co.uk/sport/0/16189337/ticker.sjson",
        feed_type: "bbc_ticker",
        show_title: true,
        news_type: "breaking",
        update_interval: 120000
    });

This will load the news items from the ticker feed on the BBC Sport website, set the feed type to "bbc_ticker" to ensure that the JSON data is interpreted correctly, set the title of the article to show (which for sport news items will be the area of sport that the article relates to) and request the JSON data to be pulled from the source URL(s) every 2 minutes.
